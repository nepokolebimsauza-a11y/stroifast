# СтройФаст (витрина)

Статический лендинг-магазин: `index.html`. Деплой на GitHub Pages через workflow (ветка `gh-pages`).

## Заявки в Telegram

На **статическом** хостинге PHP не выполняется: форма предлагает запасной путь (копирование текста и мессенджеры).

На **Apache / OSPanel** положите рядом с `index.html`:

- `telegram_notify.php` — приём POST JSON с формы;
- скопируйте `telegram_config.example.php` → `telegram_config.php`, укажите `bot_token` и `chat_id` бота;
- `telegram_config.php` не коммитьте (в `.gitignore`).

## Файлы

| Файл | Назначение |
|------|------------|
| `index.html` | Сайт |
| `index.php` | Редирект на `index.html` |
| `.htaccess` | Заголовки безопасности (Apache) |
| `telegram_notify.php` | Бэкенд заявок (опционально) |

## Заметки по UX и доступности

- Модальное окно заявки и выдвижная корзина: удержание фокуса по Tab, возврат фокуса после закрытия, блокировка прокрутки страницы.
- Тосты: для ошибок используется `role="alert"`, для остальных — `role="status"`.
- Форма: проверка через `reportValidity`, имя и телефон (≥10 цифр) до отправки на сервер; запасной сценарий с копированием текста и задержкой второго сообщения.
